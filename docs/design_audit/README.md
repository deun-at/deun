# Design Audit — Handoff v3 vs. current app

Single source of truth for the design-fidelity loop ([`../superpowers/AUDIT_LOOP.md`](../superpowers/AUDIT_LOOP.md)).
The loop reads the **Findings** below each iteration: it FIXes the highest-severity open item, or — when
none are open — runs an AUDIT to compare the live app against the v3 prototype and append new findings.

- **Design** = `design_handoff/Deun Redesign v3.dc.html`, rendered in a browser (light mode).
- **Current app** = Flutter web in Chrome via Playwright (mobile 390×844), branch `feat/v3-motion-foundation`. Web is authoritative for layout/structure/color/type/copy; final light+dark pixel sign-off stays on physical device `R5CY22DR0FK`.
- **Capture harness:** [`tools/`](tools/capture.md) (`serve.js` + `capture.md`). Test group for nav/writes: **hans**.

## Severity & format
🔥 high (wrong / unusable) · ⚠️ medium (clearly off) · 💅 low (cosmetic).
Open = `- [ ]`; done = `- [x] … ✅ <SHA>`; blocked = `⛔ blocked — <reason>`. AUDIT writes each finding as:
`- [ ] <id> · <screen> · <delta> 🔥|⚠️|💅 — <file:loc> — target: <value> — ev: compare/<x>.png`

Root cause: `SheetScaffold` painted its surface with a bare `Container`+`BoxDecoration` (color + top-30 radius); the `ListTile`s in every sheet body paint ink on the nearest Material ancestor, which sat *above* that colored box, so Flutter's `ListTile … DecoratedBox will hide ink effects` assertion fired on tap. Fix was at the component: swap the `Container` for a `Material(color, borderRadius)` so it's a real Material ancestor wrapping the content (identical idle look; ink now clips to the rounded surface). Propagates to every sheet (PaidBySheet, DateOptionsSheet, claim split picker, statistics/settings/delete-confirm). One structural test that read the old `Container`/`BoxDecoration` was updated to read the `Material`. — lib/widgets/restyle/sheet_scaffold.dart
